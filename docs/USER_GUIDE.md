## 简介

欢迎使用 SillyTavern 角色卡防盗系统！本系统旨在帮助角色卡作者保护自己的创作内容，防止未经授权的使用。

### 什么是角色卡防盗系统？

角色卡防盗系统是一个三层架构的保护方案，包括：

1. **云服务器后端** - 存储和验证密码
2. **Web 管理界面** - 供作者加密角色卡和管理密码
3. **SillyTavern 插件** - 供用户解锁加密的角色卡

### 系统特点

- ✅ **简单易用** - 作者只需上传角色卡、设置密码即可
- ✅ **安全可靠** - 使用 bcrypt 加密，支持 HTTPS 通信
- ✅ **密码版本控制** - 支持密码更新后自动重新锁定
- ✅ **多设备支持** - 可在手机、电脑、平板上管理角色卡
- ✅ **自托管支持** - 可选择使用官方服务器或部署私有服务器

## 系统概述

### 角色分类

本系统涉及两类用户：

#### 1. 角色卡作者（Author）

- 创建和分发角色卡的人
- 需要注册账号并登录 Web 管理界面
- 负责加密角色卡、设置密码、管理访问权限

#### 2. 角色卡用户（User）

- 使用角色卡的最终用户
- 需要安装 SillyTavern 防盗插件
- 需要输入正确密码才能解锁角色卡

### 工作流程

```
作者 → 上传角色卡 → 设置密码 → 下载加密后的角色卡 → 分发给用户
用户 → 导入加密角色卡 → 安装插件 → 输入密码 → 解锁使用
```

### 部署模式

#### 官方公共服务器（推荐）

- 由项目维护者部署的公共服务器
- 所有作者都可以注册使用
- 无需自己部署，开箱即用
- 类似于 GitHub、百度网盘的使用方式

#### 私有服务器（可选）

- 作者可以部署自己的私有服务器
- 完全控制自己的数据和服务
- 使用提供的一键部署脚本即可搭建
- 类似于自建 GitLab、私有 NAS

## 作者使用指南

### 第一步：注册账号

1. **访问 Web 管理界面**
   - 官方服务器：访问项目提供的官方网址（如 `https://antitheft.example.com`）
   - 私有服务器：访问您自己部署的服务器地址

2. **填写注册信息**
   - 用户名：3-50 个字符，只能包含字母、数字、下划线
   - 邮箱：有效的电子邮件地址
   - 密码：建议使用强密码（至少 8 位，包含大小写字母和数字）

3. **完成注册**
   - 点击"注册"按钮
   - 系统会自动登录并跳转到控制台

### 第二步：登录系统

如果您已经注册过账号：

1. 访问 Web 管理界面
2. 输入用户名和密码
3. 点击"登录"按钮
4. 登录成功后进入控制台

**提示**：登录状态会保持 7 天，期间无需重复登录。

### 第三步：加密角色卡

#### 3.1 准备角色卡文件

- 确保您有一个 SillyTavern 格式的角色卡 PNG 文件
- 角色卡应该是完整的、可正常导入 SillyTavern 的文件

#### 3.2 上传并加密

1. **进入角色卡列表页面**
   - 在控制台点击"角色卡管理"或"我的角色卡"

2. **点击"加密角色卡"按钮**
   - 会弹出加密对话框

3. **上传角色卡 PNG 文件**
   - 点击"选择文件"或拖拽文件到上传区域
   - 系统会自动解析角色卡信息

4. **设置角色卡信息**
   - **角色卡名称**：为这个角色卡起一个便于管理的名称（如"角色A"）
   - **密码**：设置一个密码，用户需要输入此密码才能解锁
     - 密码长度：1-100 个字符
     - 支持字母、数字、符号、中文等任意字符
     - 建议使用强密码以提高安全性

5. **点击"加密"按钮**
   - 系统会生成唯一的 Card ID（6-8 位数字）
   - 系统会自动生成防盗脚本并嵌入到角色卡中
   - 系统会显示加密成功提示

6. **下载加密后的角色卡**
   - 点击"下载"按钮保存加密后的 PNG 文件
   - 这个文件就是您要分发给用户的加密角色卡

**重要提示**：

- 角色卡 PNG 文件**不会存储在服务器上**
- 服务器只保存角色卡的元数据（Card ID、名称、密码哈希、版本号）
- 请妥善保管原始角色卡文件和加密后的文件

#### 3.3 分发加密角色卡

- 将下载的加密 PNG 文件分发给用户
- 通过您选择的方式告知用户密码（私聊、群公告等）
- 提醒用户需要安装防盗插件才能使用

### 第四步：管理角色卡

#### 4.1 查看角色卡列表

在"角色卡管理"页面，您可以看到所有已创建的角色卡：

- **Card ID**：角色卡的唯一标识符
- **角色卡名称**：您设置的名称
- **密码版本**：当前密码的版本号（每次更新密码会递增）
- **创建时间**：角色卡创建的日期和时间
- **更新时间**：最后一次修改的日期和时间

#### 4.2 编辑角色卡名称

1. 点击角色卡列表中的"编辑"按钮
2. 修改角色卡名称
3. 点击"保存"

**注意**：修改名称不会影响密码版本号。

#### 4.3 删除角色卡

1. 点击角色卡列表中的"删除"按钮
2. 确认删除操作
3. 角色卡记录将从数据库中删除

**警告**：

- 删除操作不可恢复
- 删除后，所有用户将无法验证密码（即使他们有正确的密码）
- 已解锁的实例在下次加载时会被重新锁定

### 第五步：管理密码

#### 5.1 更新密码

当您需要更改角色卡密码时（例如密码泄露）：

1. 在角色卡列表中找到目标角色卡
2. 点击"更新密码"按钮
3. 输入新密码
4. 确认新密码
5. 点击"更新"

**效果**：

- 密码版本号会自动递增（如从 1 变为 2）
- 所有已解锁的角色卡实例会在下次加载时被重新锁定
- 用户需要输入新密码才能重新解锁

#### 5.2 生成随机密码

如果您想使用强随机密码：

1. 在角色卡列表中找到目标角色卡
2. 点击"生成随机密码"按钮
3. 系统会自动生成 16 位强密码（包含大小写字母、数字、符号）
4. 密码会显示在屏幕上
5. 点击"复制"按钮复制密码
6. 密码版本号会自动递增

**提示**：

- 请立即保存生成的密码，系统不会再次显示
- 建议使用密码管理器保存密码

#### 5.3 密码版本控制说明

**什么是密码版本？**

- 每个角色卡都有一个密码版本号，初始值为 1
- 每次更新密码时，版本号会递增（1 → 2 → 3 ...）
- 用户解锁时，插件会保存当前的密码版本号

**版本控制的作用**

- 当用户下次加载角色卡时，插件会检查本地版本号和服务器版本号
- 如果版本号不匹配，说明密码已更新
- 插件会自动重新锁定角色卡，要求用户输入新密码

**使用场景**

- 密码泄露时，更新密码可以让所有已解锁的实例失效
- 定期更换密码以提高安全性
- 撤销某些用户的访问权限

## 用户使用指南

### 第一步：安装防盗插件

#### 1.1 下载插件

- 从项目官方页面下载插件压缩包（`character-card-anti-theft-plugin.zip`）
- 或从 GitHub Releases 页面下载最新版本

#### 1.2 安装插件到 SillyTavern

1. **解压插件文件**
   - 将下载的 ZIP 文件解压到临时目录

2. **找到 SillyTavern 插件目录**
   - 打开 SillyTavern 安装目录
   - 进入 `public/scripts/extensions/third-party/` 目录
   - 如果 `third-party` 目录不存在，请手动创建

3. **复制插件文件**
   - 将解压后的 `character-card-anti-theft` 文件夹复制到 `third-party` 目录
   - 最终路径应该是：`SillyTavern/public/scripts/extensions/third-party/character-card-anti-theft/`

4. **重启 SillyTavern**
   - 关闭 SillyTavern
   - 重新启动 SillyTavern

5. **验证安装**
   - 打开 SillyTavern
   - 点击顶部的"扩展"按钮（拼图图标）
   - 在扩展列表中查找"角色卡防盗"插件
   - 如果看到该插件，说明安装成功

**目录结构示例**：
```
SillyTavern/
└── public/
    └── scripts/
        └── extensions/
            └── third-party/
                └── character-card-anti-theft/
                    ├── manifest.json
                    ├── index.js
                    └── src/
                        ├── api.js
                        ├── cardManager.js
                        └── ...
```

### 第二步：导入加密的角色卡

1. **获取加密角色卡**
   - 从作者处获取加密的角色卡 PNG 文件
   - 确保您已经获得了密码

2. **导入角色卡到 SillyTavern**
   - 打开 SillyTavern
   - 点击"角色"标签
   - 点击"导入角色"按钮
   - 选择加密的 PNG 文件
   - 点击"打开"

3. **等待密码弹窗**
   - 导入后，系统会自动显示密码输入弹窗
   - 弹窗会覆盖整个屏幕，无法关闭

### 第三步：解锁角色卡

#### 3.1 正常解锁流程

1. **检查插件状态**
   - 如果弹窗显示"请安装防盗插件"，说明插件未正确安装
   - 请返回第一步重新安装插件

2. **输入密码**
   - 在密码输入框中输入作者提供的密码
   - 密码区分大小写，请准确输入

3. **提交验证**
   - 点击"验证"按钮，或按 Enter 键
   - 系统会显示"验证中..."加载提示

4. **等待验证结果**
   - 如果密码正确，弹窗会显示"验证成功！正在加载角色卡..."
   - 1 秒后弹窗自动关闭，角色卡正常加载
   - 如果密码错误，会显示错误提示，弹窗保持打开

#### 3.2 密码错误处理

如果输入密码后显示"密码错误"：

1. **检查密码**
   - 确认密码是否正确（注意大小写）
   - 确认是否有多余的空格

2. **联系作者**
   - 如果确认密码正确但仍然失败，请联系作者
   - 可能是作者更新了密码

3. **重试**
   - 重新输入正确的密码
   - 点击"验证"按钮

#### 3.3 网络错误处理

如果显示"网络错误"：

1. **检查网络连接**
   - 确保您的设备已连接到互联网
   - 尝试访问其他网站验证网络

2. **检查服务器状态**
   - 服务器可能暂时不可用
   - 等待几分钟后重试

3. **联系作者**
   - 如果问题持续存在，联系作者确认服务器地址

#### 3.4 速率限制

如果显示"请求过于频繁，请稍后再试"：

- 系统检测到短时间内多次验证请求
- 这是为了防止暴力破解攻击
- 请等待 1 分钟后再次尝试

**速率限制规则**：
- 每个 IP 地址：10 次/分钟
- 每个角色卡：5 次/分钟

### 第四步：使用解锁后的角色卡

#### 4.1 正常使用

- 解锁成功后，角色卡会正常加载
- 您可以像使用普通角色卡一样使用它
- 对话、编辑、导出等功能都可以正常使用

#### 4.2 解锁状态保持

- 解锁状态会保存在角色卡的配置中
- 下次加载同一个角色卡时，如果密码未更新，无需重新输入密码
- 解锁状态会记录解锁时间和密码版本号

#### 4.3 密码更新后的重新锁定

如果作者更新了密码：

1. **自动检测**
   - 下次加载角色卡时，插件会自动检查密码版本
   - 如果版本不匹配，角色卡会被重新锁定

2. **重新验证**
   - 系统会显示密码输入弹窗
   - 弹窗会提示"作者已更新密码，请重新输入"
   - 输入新密码即可重新解锁

### 第五步：故障排查

#### 问题 1：弹窗显示"请安装防盗插件"

**原因**：插件未正确安装或未加载

**解决方法**：
1. 检查插件是否安装在正确的目录
2. 重启 SillyTavern
3. 在扩展列表中确认插件已启用
4. 检查浏览器控制台是否有错误信息

#### 问题 2：弹窗无法关闭

**原因**：这是正常行为，弹窗设计为不可关闭

**解决方法**：
- 唯一关闭弹窗的方法是输入正确的密码并验证成功
- 如果您没有密码，请联系作者获取
- 如果您不想使用该角色卡，可以删除它

#### 问题 3：密码正确但验证失败

**可能原因**：
1. 密码输入有误（大小写、空格）
2. 作者已更新密码
3. 服务器地址配置错误
4. 网络连接问题

**解决方法**：
1. 仔细检查密码输入
2. 联系作者确认当前密码
3. 检查网络连接
4. 查看浏览器控制台的错误信息

#### 问题 4：Card ID 不存在

**原因**：服务器上没有该角色卡的记录

**可能情况**：
1. 作者删除了该角色卡记录
2. 角色卡配置错误
3. 连接到了错误的服务器

**解决方法**：
- 联系作者确认角色卡状态
- 请求作者重新创建角色卡记录

#### 问题 5：验证速度很慢

**可能原因**：
1. 网络延迟
2. 服务器负载高
3. 服务器地址配置错误

**解决方法**：
1. 检查网络连接速度
2. 等待服务器负载降低
3. 联系作者确认服务器状态

## 常见问题解答

### 关于系统

#### Q1：这个系统是如何工作的？

**A**：系统分为三个部分：

1. **云服务器**：存储角色卡的元数据（Card ID、密码哈希、版本号）
2. **Web 管理界面**：作者用来加密角色卡和管理密码
3. **SillyTavern 插件**：用户用来验证密码和解锁角色卡

当用户导入加密角色卡时，嵌入的脚本会显示密码弹窗，插件会将密码发送到服务器验证，验证成功后解锁角色卡。

#### Q2：角色卡 PNG 文件会上传到服务器吗？

**A**：不会。角色卡 PNG 文件**不会存储在服务器上**。

- 加密流程：上传 PNG → 解析元数据 → 生成脚本 → 嵌入脚本 → 下载加密后的 PNG
- 服务器只保存元数据（Card ID、名称、密码哈希、版本号）
- PNG 文件始终在您的本地设备上

#### Q3：密码是如何存储的？

**A**：密码使用 bcrypt 算法加密后存储在数据库中。

- bcrypt 是一种单向加密算法，无法解密
- 即使数据库泄露，攻击者也无法获取明文密码
- 验证时使用 bcrypt.compare() 比较密码哈希

#### Q4：系统支持哪些设备？

**A**：

- **Web 管理界面**：支持所有现代浏览器（Chrome、Firefox、Safari、Edge）
- **设备**：电脑、手机、平板都可以访问 Web 界面
- **SillyTavern 插件**：需要在运行 SillyTavern 的设备上安装

#### Q5：可以离线使用吗？

**A**：不可以。系统需要联网才能验证密码。

- 密码验证需要连接到服务器
- 密码版本检查需要联网
- 但解锁后的角色卡可以离线使用（直到下次加载）

### 关于作者功能

#### Q6：如何加密角色卡？

**A**：请参考 [作者使用指南 - 第三步：加密角色卡](#第三步加密角色卡)

简要步骤：
1. 登录 Web 管理界面
2. 点击"加密角色卡"
3. 上传 PNG 文件
4. 设置名称和密码
5. 下载加密后的文件

#### Q7：可以为同一个角色卡创建多个密码吗？

**A**：不可以。每个角色卡只能有一个密码。

- 如果需要不同的密码，可以创建多个角色卡记录
- 每个记录会有不同的 Card ID
- 需要分别加密和分发

#### Q8：忘记密码怎么办？

**A**：作为作者，您可以随时更新密码。

1. 登录 Web 管理界面
2. 找到对应的角色卡
3. 点击"更新密码"
4. 设置新密码
5. 将新密码告知用户

**注意**：更新密码后，所有已解锁的实例会被重新锁定。

#### Q9：可以撤销某个用户的访问权限吗？

**A**：可以通过更新密码来实现。

1. 更新角色卡密码
2. 只将新密码告知授权用户
3. 未授权用户将无法通过验证

**限制**：无法针对单个用户撤销，只能通过更换密码影响所有用户。

#### Q10：删除角色卡记录会怎样？

**A**：删除后：

- 服务器上的元数据会被永久删除
- 所有用户（包括已解锁的）都无法验证密码
- 已解锁的实例在下次加载时会被重新锁定
- 删除操作不可恢复

**建议**：除非确定不再需要，否则不要删除角色卡记录。

#### Q11：可以导出角色卡列表吗？

**A**：目前不支持导出功能。

- 您可以在 Web 界面查看所有角色卡
- 可以手动记录 Card ID 和密码
- 建议使用密码管理器保存密码

### 关于用户功能

#### Q12：必须安装插件吗？

**A**：是的，必须安装插件才能解锁加密的角色卡。

- 插件提供密码验证功能
- 插件负责与服务器通信
- 没有插件，密码输入框会被禁用

#### Q13：插件会收集我的数据吗？

**A**：不会。插件只会：

- 发送 Card ID 和密码到服务器进行验证
- 保存解锁状态到角色卡配置中
- 不会收集任何其他数据

#### Q14：解锁后可以导出角色卡吗？

**A**：可以。解锁后的角色卡可以正常导出。

- 导出的文件会保留防盗配置
- 导出的文件仍然是加密的
- 其他人导入后仍需要密码

#### Q15：可以修改解锁后的角色卡吗？

**A**：可以。解锁后可以正常编辑角色卡。

- 编辑不会影响防盗配置
- 保存后仍然需要密码才能解锁
- 防盗配置会保留在角色卡中

#### Q16：弹窗为什么无法关闭？

**A**：这是设计行为，用于确保角色卡的安全性。

- 弹窗禁用了所有关闭方式（ESC、点击外部、关闭按钮）
- 唯一关闭方式是输入正确密码并验证成功
- 这样可以防止未授权访问

#### Q17：密码输入错误次数有限制吗？

**A**：有速率限制，但没有次数限制。

- 每个 IP：10 次/分钟
- 每个 Card ID：5 次/分钟
- 超过限制需要等待 1 分钟
- 不会永久锁定账号

#### Q18：作者更新密码后，我需要重新下载角色卡吗？

**A**：不需要。

- 角色卡文件本身不需要更新
- 只需要输入新密码即可
- 系统会自动检测密码版本并要求重新验证

### 关于安全性

#### Q19：系统安全吗？

**A**：系统采用了多种安全措施：

- ✅ 密码使用 bcrypt 加密（salt rounds = 12）
- ✅ 支持 HTTPS 加密通信
- ✅ JWT 认证（7 天有效期）
- ✅ 速率限制防止暴力破解
- ✅ 输入验证防止注入攻击
- ✅ 参数化查询防止 SQL 注入

#### Q20：密码会被明文传输吗？

**A**：在生产环境中不会。

- 生产环境强制使用 HTTPS
- 密码在传输过程中会被加密
- 服务器收到后立即进行 bcrypt 验证
- 明文密码不会被存储

#### Q21：服务器管理员能看到我的密码吗？

**A**：不能。

- 密码使用 bcrypt 单向加密
- 数据库中只存储密码哈希
- 即使是管理员也无法解密
- 日志系统不会记录明文密码

#### Q22：如果服务器被攻击怎么办？

**A**：系统有多层防护：

1. **密码加密**：即使数据库泄露，密码也无法解密
2. **速率限制**：防止暴力破解
3. **JWT 认证**：防止未授权访问
4. **日志记录**：记录所有关键操作

**建议**：
- 使用强密码
- 定期更换密码
- 使用官方服务器或可信的私有服务器

### 关于私有服务器

#### Q23：如何部署私有服务器？

**A**：请参考 [部署文档](DEPLOYMENT.md)。

系统提供两种一键部署方式：
1. **Docker 部署**（推荐）：适用于 Linux、macOS、Windows
2. **Windows 原生部署**：适用于 Windows 10/11

#### Q24：私有服务器和官方服务器有什么区别？

**A**：功能完全相同，区别在于：

| 特性 | 官方服务器       | 私有服务器         |
| ---- | ---------------- | ------------------ |
| 部署 | 无需部署         | 需要自己部署       |
| 维护 | 由项目维护者负责 | 需要自己维护       |
| 数据 | 存储在官方服务器 | 存储在自己的服务器 |
| 成本 | 免费使用         | 需要服务器成本     |
| 控制 | 有限             | 完全控制           |

#### Q25：私有服务器需要什么配置？

**A**：最低配置：

- **CPU**：1 核
- **内存**：1 GB
- **存储**：10 GB
- **网络**：公网 IP 或域名
- **操作系统**：Linux（推荐 Ubuntu 20.04+）或 Windows 10/11

**推荐配置**：

- **CPU**：2 核
- **内存**：2 GB
- **存储**：20 GB
- **带宽**：5 Mbps

#### Q26：私有服务器可以只供自己使用吗？

**A**：可以。

- 您可以只为自己的角色卡创建记录
- 不需要开放注册功能
- 可以通过防火墙限制访问

## 故障排查

### 作者端问题

#### 问题 1：无法登录 Web 管理界面

**症状**：输入用户名和密码后显示"用户名或密码错误"

**可能原因**：
1. 用户名或密码输入错误
2. 账号不存在
3. 服务器连接问题

**解决步骤**：

1. **检查输入**
   - 确认用户名和密码是否正确
   - 注意大小写
   - 检查是否有多余的空格

2. **重置密码**
   - 如果忘记密码，联系服务器管理员
   - 或使用"忘记密码"功能（如果已实现）

3. **检查网络**
   - 确认能否访问服务器地址
   - 尝试访问 `/health` 端点检查服务器状态

4. **查看浏览器控制台**
   - 按 F12 打开开发者工具
   - 查看 Console 和 Network 标签
   - 查找错误信息

#### 问题 2：上传角色卡后无法解析

**症状**：上传 PNG 文件后显示"无法解析角色卡"

**可能原因**：
1. 文件不是有效的 SillyTavern 角色卡
2. 文件损坏
3. 文件格式不正确

**解决步骤**：

1. **验证文件**
   - 确认文件是 PNG 格式
   - 尝试在 SillyTavern 中导入该文件
   - 确认文件包含角色卡元数据

2. **重新导出**
   - 在 SillyTavern 中重新导出角色卡
   - 使用新导出的文件重试

3. **检查文件大小**
   - 确认文件大小合理（通常 < 10 MB）
   - 过大的文件可能导致解析失败

#### 问题 3：加密后的角色卡无法下载

**症状**：点击"下载"按钮后没有反应

**可能原因**：
1. 浏览器阻止了下载
2. JavaScript 错误
3. 文件生成失败

**解决步骤**：

1. **检查浏览器设置**
   - 确认浏览器允许下载
   - 检查是否有弹窗拦截

2. **查看控制台**
   - 按 F12 打开开发者工具
   - 查看是否有 JavaScript 错误

3. **重试**
   - 刷新页面
   - 重新加密角色卡

#### 问题 4：更新密码后用户仍能使用旧密码

**症状**：更新密码后，用户使用旧密码仍能解锁

**可能原因**：
1. 用户的角色卡已经解锁（本地缓存）
2. 密码版本未正确更新
3. 服务器缓存问题

**解决步骤**：

1. **检查密码版本**
   - 在角色卡列表中确认密码版本号已递增
   - 如果未递增，说明更新失败

2. **清除缓存**
   - 如果服务器启用了缓存，清除缓存
   - 重启服务器

3. **通知用户**
   - 告知用户需要重新加载角色卡
   - 已解锁的实例需要重新导入才会检测版本

### 用户端问题

## 角色卡配置示例

### 加密前的角色卡结构

```json
{
  "name": "示例角色",
  "description": "这是一个示例角色",
  "first_mes": "你好！",
  "extensions": {
    // 其他扩展配置
  }
}
```

### 加密后的角色卡结构

```json
{
  "name": "示例角色",
  "description": "这是一个示例角色",
  "first_mes": "你好！",
  "extensions": {
    "anti_theft": {
      "enabled": true,
      "card_id": "123456",
      "server_url": "https://antitheft.example.com",
      "locked": true,
      "password_version": 1
    },
    "anti_theft_script": "(function(){...})();"
  }
}
```

### 解锁后的角色卡结构

```json
{
  "name": "示例角色",
  "description": "这是一个示例角色",
  "first_mes": "你好！",
  "extensions": {
    "anti_theft": {
      "enabled": true,
      "card_id": "123456",
      "server_url": "https://antitheft.example.com",
      "locked": false,
      "password_version": 1,
      "last_unlock_time": 1704067200
    },
    "anti_theft_script": "(function(){...})();"
  }
}
```

### 配置字段说明

| 字段               | 类型    | 必需 | 说明                          |
| ------------------ | ------- | ---- | ----------------------------- |
| `enabled`          | boolean | 是   | 是否启用防盗功能              |
| `card_id`          | string  | 是   | 角色卡唯一标识符（6-8位数字） |
| `server_url`       | string  | 是   | 验证服务器地址                |
| `locked`           | boolean | 是   | 当前锁定状态                  |
| `password_version` | number  | 否   | 密码版本号                    |
| `last_unlock_time` | number  | 否   | 上次解锁时间（Unix 时间戳）   |
