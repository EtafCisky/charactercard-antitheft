# SillyTavern 角色卡防盗系统

## 这是什么？

这是一个保护 SillyTavern 角色卡的系统，可以防止您精心制作的角色卡被他人盗用。就像给您的角色卡加了一把锁，只有输入正确的密码才能使用。

**📚 不确定系统架构？** 查看详细说明：

- [系统架构说明](../.kiro/specs/character-card-anti-theft/SYSTEM_ARCHITECTURE.md) - 文字详解
- [架构可视化图](../.kiro/specs/character-card-anti-theft/ARCHITECTURE_VISUAL.md) - 图形化展示

## 系统架构

本系统采用 **SaaS（Software as a Service）模式**，类似于 GitHub、百度网盘的使用方式：

### 🌐 作者公共服务器（推荐使用方式）

- **一个人部署，所有人使用**
- 项目作者部署一个公共服务器（如 <http://154.9.228.252）>
- 所有角色卡作者通过浏览器访问该网站
- 支持任何设备：手机、电脑、平板、任何浏览器
- 每个作者注册独立账号，管理自己的角色卡
- **无需自己部署服务器，注册即用**

### 🏠 私有服务器（可选部署方式）

- 作者也可以选择部署自己的私有服务器
- 完全控制自己的数据和服务
- 使用提供的一键部署脚本即可搭建

## 使用流程

### 对于角色卡使用者（普通用户）

1. 下载并安装 SillyTavern 插件
2. 导入加密的角色卡
3. 输入密码解锁
4. 完成！

### 对于角色卡作者

#### 使用作者服务器（推荐）

1. 访问作者网站：<http://154.9.228.252>
2. 注册账号（可以在手机、电脑、平板等任何设备上注册）
3. 登录后上传角色卡 PNG 文件
4. 设置密码
5. 下载加密后的角色卡
6. 分发给用户
7. 随时可以在任何设备上登录管理密码

#### 部署私有服务器（可选）

如果你想要完全控制自己的数据：

1. 下载部署脚本
2. 运行一键安装
3. 访问自己的服务器地址
4. 其他步骤同上

## 为什么需要这个系统？

- **保护创作成果**：您花费大量时间创作的角色卡值得被保护
- **防止盗用**：防止他人未经许可使用或修改您的角色卡
- **简单易用**：提供友好的管理界面，无需编程知识
- **多设备支持**：可以在任何设备上管理您的角色卡
- **随时随地**：只要有网络，就能管理密码

## 我应该选择哪种部署方式？

**注意：** 这些部署方式是给**想要搭建服务器的人**使用的。

- 如果你是**角色卡作者**，只想使用系统，**不需要部署**，直接访问作者网站（<http://154.9.228.252）注册即可>
- 如果你想**搭建公共服务器**（供所有人使用），或者**搭建私有服务器**（只给自己用），才需要选择部署方式

我们提供两种部署方式，请根据您的情况选择：

### 方式一：Docker 部署（推荐新手）

**适合人群**：

- 想要搭建公共服务器或私有服务器的人
- 完全不懂技术的新手
- 想要最简单的安装方式
- 使用 Windows、macOS 或 Linux 系统

**优点**：

- ✅ 一键安装，自动配置所有环境
- ✅ 不需要手动安装 Node.js 和 MySQL
- ✅ 出问题容易恢复
- ✅ 跨平台，任何系统都能用

**缺点**：

- ❌ 需要下载 Docker（约 500MB）
- ❌ 占用资源稍多一些

**👉 [查看 Docker 部署教程](./docker/README.md)**

### 方式二：Windows 原生部署

**适合人群**：

- 想要搭建服务器的人
- 只使用 Windows 系统
- 想要最佳性能
- 愿意手动安装一些软件

**优点**：

- ✅ 性能最好，运行最快
- ✅ 占用资源少
- ✅ 便于调试和查看日志

**缺点**：

- ❌ 需要手动安装 Node.js 和 MySQL
- ❌ 配置步骤稍多一些
- ❌ 仅支持 Windows 系统

**👉 [查看 Windows 部署教程](./windows/README.md)**

## 还不确定选哪个？

**如果你是角色卡作者，只想使用系统：**

- ❌ 不需要部署服务器
- ✅ 直接访问官方网站注册账号即可
- ✅ 可以在手机、电脑、平板等任何设备上使用

**如果你想搭建服务器：**

- 想要最简单的方式 → 选择 **Docker 部署**
- 想要最好的性能 → 选择 **Windows 原生部署**
