# 角色卡加密功能实现文档

## 概述

本文档描述了任务 15.2 "实现角色卡加密功能" 的完整实现。该功能允许作者在 Web 管理界面上传角色卡 PNG 文件，系统自动生成并嵌入防盗脚本，然后提供加密后的 PNG 文件下载。

## 实现的功能

### 1. PNG 元数据处理 (`src/utils/pngMetadata.js`)

**功能：**
- 从 PNG 文件中读取 SillyTavern 角色卡元数据
- 将修改后的元数据写回 PNG 文件
- 支持 tEXt chunk 格式（keyword: "chara"）
- 自动处理 base64 编码/解码

**关键函数：**
- `readPngMetadata(file)` - 读取 PNG 中的角色卡 JSON 数据
- `writePngMetadata(file, metadata)` - 将修改后的元数据写回 PNG

**技术细节：**
- 使用 FileReader API 读取二进制数据
- 解析 PNG chunk 结构
- 计算 CRC32 校验和
- 保持 PNG 文件完整性

### 2. 防盗脚本生成器 (`src/utils/scriptGenerator.js`)

**功能：**
- 生成嵌入到角色卡中的防盗脚本
- 创建不可关闭的密码弹窗
- 实现插件检测逻辑
- 实现密码验证调用逻辑

**关键函数：**
- `generateAntiTheftScript(cardId, serverUrl)` - 生成完整的 IIFE 脚本

**脚本特性：**
- 不可关闭的全屏遮罩弹窗
- 禁用 ESC 键、背景点击等关闭方式
- 自动检测防盗插件是否安装
- 未安装插件时禁用密码输入
- 密码验证成功是唯一关闭弹窗的方式
- 美观的 UI 设计（使用内联样式）

### 3. 加密角色卡模态框 (`src/components/EncryptCardModal.jsx`)

**功能：**
- 三步式加密流程（上传 → 配置 → 完成）
- PNG 文件上传和验证
- 角色卡名称和密码配置
- 创建角色卡记录 API 调用
- 生成并嵌入防盗脚本
- 提供加密后的 PNG 下载

**步骤详解：**

#### 步骤 1: 上传角色卡
- 拖拽或点击上传 PNG 文件
- 验证文件类型（必须是 PNG）
- 解析角色卡元数据
- 自动填充角色卡名称
- 显示解析成功提示

#### 步骤 2: 配置加密
- 输入角色卡名称（必填）
- 输入密码（必填，1-100 字符）
- 确认密码（必须匹配）
- 显示当前服务器地址
- 表单验证

#### 步骤 3: 完成
- 显示加密成功提示
- 展示 Card ID、密码版本、服务器地址
- 提供下载按钮
- 显示重要提示信息

### 4. CardList 页面集成

**修改内容：**
- 添加 "加密角色卡" 按钮
- 集成 EncryptCardModal 组件
- 加密成功后自动刷新列表

## 工作流程

```
1. 用户点击 "加密角色卡" 按钮
   ↓
2. 打开模态框，上传 PNG 文件
   ↓
3. 系统解析 PNG 元数据
   ↓
4. 用户输入角色卡名称和密码
   ↓
5. 系统调用 API 创建角色卡记录
   ↓
6. 系统生成防盗脚本（包含 card_id 和 server_url）
   ↓
7. 系统将脚本嵌入到 extensions.anti_theft_script
   ↓
8. 系统设置 extensions.anti_theft 配置
   ↓
9. 系统将修改后的元数据写回 PNG
   ↓
10. 用户下载加密后的 PNG 文件
```

## 关键设计决策

### 1. server_url 来源
**决策：** 使用 `apiClient.defaults.baseURL` 作为 server_url

**原因：**
- 确保用户验证时连接到作者使用的服务器
- 支持多租户和自托管部署
- 如果作者使用官方服务器，用户也连接官方服务器
- 如果作者使用自己的服务器，用户连接作者的服务器

### 2. PNG 文件不存储在服务器
**决策：** 只存储元数据（card_id、名称、密码哈希、版本号）

**原因：**
- 减少服务器存储压力
- 保护作者隐私（角色卡内容不上传）
- 简化系统架构
- 加密流程完全在客户端完成

### 3. 三步式流程
**决策：** 分为上传、配置、完成三个步骤

**原因：**
- 清晰的用户体验
- 每步专注一个任务
- 便于错误处理和验证
- 提供明确的进度反馈

### 4. 不可关闭的弹窗
**决策：** 嵌入脚本创建的弹窗无法通过任何方式关闭（除了密码验证成功）

**原因：**
- 强制用户安装插件或输入密码
- 防止绕过验证机制
- 符合防盗需求
- 提供清晰的错误提示

## API 调用

### POST /api/cards
**请求：**
```json
{
  "card_name": "角色卡名称",
  "password": "用户设置的密码"
}
```

**响应：**
```json
{
  "success": true,
  "card_id": "123456",
  "password_version": 1,
  "message": "角色卡创建成功"
}
```

## 元数据结构

### 修改前
```json
{
  "data": {
    "name": "角色名称",
    "description": "...",
    "extensions": {
      // 其他扩展
    }
  }
}
```

### 修改后
```json
{
  "data": {
    "name": "角色名称",
    "description": "...",
    "extensions": {
      "anti_theft": {
        "enabled": true,
        "card_id": "123456",
        "server_url": "http://localhost:3000",
        "locked": true,
        "password_version": 1
      },
      "anti_theft_script": "(function() { ... })();"
    }
  }
}
```

## 错误处理

### 文件上传错误
- 非 PNG 文件：显示 "请上传 PNG 格式的角色卡文件"
- 解析失败：显示 "解析角色卡失败，请确保这是有效的 SillyTavern 角色卡文件"
- 缺少元数据：显示 "角色卡格式错误：缺少 data 字段"

### 表单验证错误
- 角色卡名称为空：显示 "请输入角色卡名称"
- 密码为空：显示 "请输入密码"
- 密码不匹配：显示 "两次输入的密码不一致"
- 密码长度错误：显示 "密码长度必须在 1-100 字符之间"

### API 错误
- 网络错误：显示 "网络错误，请稍后重试"
- 服务器错误：显示服务器返回的错误消息
- 创建失败：显示 "创建角色卡记录失败"

## 已知限制

1. **浏览器兼容性**
   - 需要支持 FileReader API
   - 需要支持 Blob API
   - 需要支持 atob/btoa（base64）

2. **文件大小**
   - 大文件可能导致浏览器内存占用较高
   - 建议限制在 10MB 以内

3. **PNG 格式**
   - 仅支持标准 PNG 格式
   - 必须包含 tEXt chunk
   - 必须有 "chara" keyword
